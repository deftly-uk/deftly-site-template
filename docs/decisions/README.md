# Architecture Decision Log

Short records of significant, hard-to-reverse decisions: the problem, the options we
weighed, what we chose, and why. The point is that when a decision "comes back to bite"
later, the reasoning is written down rather than lost.

One file per decision, numbered in order. Keep them brief. A decision can be superseded by
a later one (note it at the top of the old record and link the new one).

The big day-to-day rules of the codebase live in [`CONSTITUTION.md`](../../CONSTITUTION.md);
this log is for the "why we went this way and not that way" calls behind them.
